
# 三、基数排序（Radix Sort）知识点大全 ⭐

## 1️⃣ 核心思想（必背）

> 按位排序，从低位到高位（LSD）或从高位到低位（MSD）

📌 **最常考：LSD（低位优先）**

---

## 2️⃣ 为什么基数排序“不是比较排序”？

👉 它**不比较大小，只按位分组**

这句话在面试中属于**加分点**。

---

## 3️⃣ 算法流程（LSD）

以 `[170, 45, 75, 90]` 为例：

1. 按 **个位** 分桶 → 合并
2. 按 **十位** 分桶 → 合并
3. 按 **百位** 分桶 → 合并

⚠️ 每一轮分桶排序**必须是稳定的**

---

## 4️⃣ 时间 & 空间复杂度

| 项  | 复杂度                |
| -- | ------------------ |
| 时间 | **O(d × (n + k))** |
| 空间 | O(n + k)           |

* d：最大位数
* k：基数（通常 10）

📌 当 d 较小、n 很大 → 非常快

---

## 5️⃣ 稳定性

✅ **稳定**（前提：每一轮桶内是稳定的）

---

## 6️⃣ 适用场景

✅ 适合：

* 整数
* 字符串（字典序）
* 位数固定或差异不大

❌ 不适合：

* 浮点数
* 位数极度不统一
* 负数（需额外处理）

---

## 7️⃣ JS 实现（LSD 版本）

```js
function radixSort(arr) {
  const maxDigit = getMaxDigit(arr);

  for (let d = 0; d < maxDigit; d++) {
    const buckets = Array.from({ length: 10 }, () => []);

    for (let num of arr) {
      const digit = Math.floor(num / Math.pow(10, d)) % 10;
      buckets[digit].push(num);
    }

    arr = [].concat(...buckets);
  }

  return arr;
}

function getMaxDigit(arr) {
  return Math.max(...arr).toString().length;
}
```

## 8️⃣ 面试高频问法

### Q1：为什么基数排序必须稳定？

👉 因为高位排序必须保留低位排序结果

### Q2：基数排序和桶排序的本质区别？

👉

* 桶排序：按 **区间**
* 基数排序：按 **位**

### Q3：为什么不能直接排序负数？

👉 位运算无法直接处理符号位，需要拆分正负

# 四、桶排序 vs 基数排序（对比表，面试必背）

| 对比项  | 桶排序  | 基数排序     |
| ---- | ---- | -------- |
| 分类依据 | 值的范围 | 数字的位     |
| 是否比较 | 否    | 否        |
| 稳定性  | 可稳定  | 稳定       |
| 适合数据 | 分布均匀 | 位数固定     |
| 常见场景 | 浮点数  | 整数 / 字符串 |

---

# 五、面试一句话总结模板（直接用）

> **桶排序**通过值域划分区间，适合分布均匀的数据；
> **基数排序**通过位数分组，适合位数有限的整数或字符串；
> 两者都属于**非比较排序，空间换时间**。
