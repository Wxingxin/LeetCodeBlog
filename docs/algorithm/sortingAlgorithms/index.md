
# JavaScript 中的排序算法概览

JavaScript 中的排序算法主要包括基础的冒泡、选择、插入排序，进阶的希尔、归并、快速、堆排序，以及针对特定数据（如整数）的高效算法（计数、桶、基数排序），此外还有内置的 `Array.prototype.sort()` 方法。  
它们在**时间复杂度、空间复杂度以及稳定性**方面各不相同，适用于不同的使用场景。

---

## 一、简单排序（O(n²)）

### 1. 冒泡排序（Bubble Sort）
- 重复比较相邻元素，若顺序错误则交换
- 思想直观、实现简单
- 缺点：效率低，适合教学或小规模数据

### 2. 选择排序（Selection Sort）
- 每次从未排序部分中选出最小（或最大）元素
- 放到已排序部分的末尾
- 交换次数少，但整体效率仍较低

### 3. 插入排序（Insertion Sort）
- 构建有序序列
- 将未排序元素插入到合适的位置

**插入排序的变体：**
- **直接插入排序**：最基础形式
- **折半插入排序**：使用二分查找定位插入位置，提高查找效率

---

## 二、高级排序（O(n log n)）

### 1. 希尔排序（Shell Sort）
- 插入排序的改进版
- 通过分组插入减少移动次数
- 效率高于简单插入排序，但不稳定

### 2. 归并排序（Merge Sort）
- 采用分治思想
- 将数组拆分成两半递归排序，再合并
- 稳定，但需要额外空间

### 3. 快速排序（Quick Sort）
- 分治策略
- 选择一个基准值，将数组分区后递归排序
- 平均性能优秀，是最常用的排序算法之一

### 4. 堆排序（Heap Sort）
- 基于堆数据结构（最大堆或最小堆）
- 原地排序，空间效率高
- 不稳定

---

## 三、线性 / 特殊排序（O(n) 或 O(n + k)）

### 1. 计数排序（Counting Sort）
- 适用于整数数据
- 统计元素出现次数后构建有序数组
- 对数据范围敏感

### 2. 桶排序（Bucket Sort）
- 将数据分配到有限数量的桶中
- 对桶内元素再进行排序
- 适合数据分布较均匀的场景

### 3. 基数排序（Radix Sort）
- 按位进行排序（个位、十位、百位等）
- 可分为 LSD（低位优先）或 MSD（高位优先）
- 适合多位整数或字符串

---

## 四、JavaScript 内置排序

### Array.prototype.sort()
- 使用灵活
- 默认按**字符串 Unicode 顺序**比较
- 通常需要传入比较函数来自定义排序规则

---

## 五、排序算法的稳定性

### 稳定算法
- 冒泡排序
- 插入排序
- 归并排序
- 计数排序
- 基数排序  

> 相同元素的相对顺序不会改变

### 不稳定算法
- 选择排序
- 希尔排序
- 快速排序
- 堆排序  

> 相同元素的相对顺序可能发生变化

---

## 如何选择排序算法？

- **小规模数据 / 代码简单优先**  
  - 冒泡排序、插入排序、选择排序

- **通用场景（O(n log n)）**  
  - 快速排序：平均性能最好  
  - 归并排序：稳定性要求高  
  - 堆排序：空间效率高

- **特定数据类型（如整数）**  
  - 计数排序 / 桶排序 / 基数排序  
  - 在合适条件下效率远超 O(n log n)
