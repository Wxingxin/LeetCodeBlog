整数数组 nums 按升序排列，数组中的值 互不相同 。

在传递给函数之前，nums 在预先未知的某个下标 k（0 <= k < nums.length）上进行了 向左旋转，使数组变为 [nums[k], nums[k+1], ..., nums[n-1], nums[0], nums[1], ..., nums[k-1]]（下标 从 0 开始 计数）。例如， [0,1,2,4,5,6,7] 下标 3 上向左旋转后可能变为 [4,5,6,7,0,1,2] 。

给你 旋转后 的数组 nums 和一个整数 target ，如果 nums 中存在这个目标值 target ，则返回它的下标，否则返回 -1 。

你必须设计一个时间复杂度为 O(log n) 的算法解决此问题。

 

示例 1：

输入：nums = [4,5,6,7,0,1,2], target = 0
输出：4
示例 2：

输入：nums = [4,5,6,7,0,1,2], target = 3
输出：-1
示例 3：

输入：nums = [1], target = 0
输出：-1
# 

这题是 **旋转排序数组二分查找**（LeetCode 33），本质仍然是二分，但**每一步要先判断哪一半是“有序的”**。

我按 **“规则 → 示例 → 代码”** 三步给你彻底讲清楚。

---

## 一、核心规则（必须背）

在旋转数组中，**任意一个 mid**，左右两边 **至少有一边是有序的**。

判断方法：

* 如果 `nums[left] <= nums[mid]`
  👉 **左半段有序**
* 否则
  👉 **右半段有序**

---

## 二、在有序的一半里判断 target 在不在

### 情况 1：左半段有序

```
nums[left] ... nums[mid]   （递增）
```

* 如果：

  ```txt
  nums[left] <= target < nums[mid]
  ```

  👉 target 在左边 → `right = mid - 1`
* 否则 👉 去右边 → `left = mid + 1`

---

### 情况 2：右半段有序

```
nums[mid] ... nums[right]  （递增）
```

* 如果：

  ```txt
  nums[mid] < target <= nums[right]
  ```

  👉 target 在右边 → `left = mid + 1`
* 否则 👉 去左边 → `right = mid - 1`

---

## 三、结合示例一步一步走

```txt
nums = [4,5,6,7,0,1,2]
target = 0
```

---

### 第 1 轮

```
left=0, right=6
mid=3 → nums[mid]=7
```

判断有序：

```
nums[left]=4 <= nums[mid]=7  → 左半段有序
```

target 在左半段吗？

```
4 <= 0 < 7 ❌
```

👉 去右边：

```
left = mid + 1 = 4
```

---

### 第 2 轮

```
left=4, right=6
mid=5 → nums[mid]=1
```

判断有序：

```
nums[left]=0 <= nums[mid]=1 → 左半段有序
```

target 在左半段吗？

```
0 <= 0 < 1 ✅
```

👉 去左边：

```
right = mid - 1 = 4
```

---

### 第 3 轮

```
left=4, right=4
mid=4 → nums[mid]=0
```

命中 🎯
返回 `4`

---

## 四、代码（可直接提交 JS）

```js
/**
 * @param {number[]} nums
 * @param {number} target
 * @return {number}
 */
var search = function(nums, target) {
  let left = 0;
  let right = nums.length - 1;

  while (left <= right) {
    const mid = left + ((right - left) >> 1);

    if (nums[mid] === target) return mid;

    // 左半段有序
    if (nums[left] <= nums[mid]) {
      if (nums[left] <= target && target < nums[mid]) {
        right = mid - 1;
      } else {
        left = mid + 1;
      }
    }
    // 右半段有序
    else {
      if (nums[mid] < target && target <= nums[right]) {
        left = mid + 1;
      } else {
        right = mid - 1;
      }
    }
  }

  return -1;
};
```

---

## 五、为什么是 O(log n)？

* 每一轮都**排除一半区间**
* 不管怎么旋转，本质还是二分

---

## 六、一句话口诀（面试必背）

> **先看哪边有序，再看 target 在不在这边**
