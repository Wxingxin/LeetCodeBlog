# 除自身以外数组的乘积

给你一个整数数组 `nums`，返回 数组 `answer` ，其中 `answer[i]` 等于`nums `中除`nums[i]`**之外**其余**各元素**的**乘积**

题目数据 **保证** 数组`nums`之中任意元素的全部**前缀元素**和**后缀的乘积**都在`32`位 **整数范围**内。

请 **不要使用除法**，且在 **O(n) 时间**复杂度内完成此题。

## 示例 1:

输入: `nums = [1,2,3,4]`

输出: `[24,12,8,6]`

## 示例 2:

输入: `nums = [-1,1,0,-3,3]`

输出: `[0,0,9,0,0]`

# 解

#### 整体思路

- **为什么不用除法？** 题目禁止使用除法，同时需要避免潜在的零除问题（如果数组有零）。
- **核心方法：前缀乘积 + 后缀乘积**
  - 第一遍遍历（从左到右）：计算每个位置的前缀乘积（即当前位置左侧所有元素的乘积），并存入 `answer` 数组。
  - 第二遍遍历（从右到左）：使用一个变量 `right` 维护后缀乘积（当前位置右侧所有元素的乘积），边遍历边乘到 `answer` 中。
- **时间复杂度**：两次 O(n) 遍历，总 O(n)。
- **空间复杂度**：O(1) 额外空间（不计输出数组）。
- **注意事项**：处理包含零或负数的数组时，乘积会自然为零或带负号；数组长度至少为 2（题目保证）。

#### 代码逐行分析

```javascript
/**
 * @param {number[]} nums
 * @return {number[]}
 */
var productExceptSelf = function (nums) {
  const n = nums.length; // 获取数组长度
  const answer = new Array(n).fill(1); // 初始化 answer 为全 1 数组，作为前缀乘积的起点

  // 第一遍：计算前缀乘积
  for (let i = 1; i < n; i++) {
    answer[i] = answer[i - 1] * nums[i - 1]; // answer[i] = nums[0] * ... * nums[i-1]
  }

  // 第二遍：计算后缀乘积，使用一个变量从右到左累乘
  let right = 1; // 初始化右后缀为 1（最右边无元素）
  for (let i = n - 1; i >= 0; i--) {
    answer[i] *= right; // 乘上右侧乘积：answer[i] = 前缀 * 后缀
    right *= nums[i]; // 更新 right 为下一个位置的后缀（right = right * nums[i]）
  }

  return answer; // 返回结果数组
};
```

#### 结合案例学习：nums = [1, 2, 3, 4]

让我们用这个简单数组逐步模拟执行过程，帮助理解代码的运行逻辑。预期输出为 [24, 12, 8, 6]（因为 2*3*4=24, 1*3*4=12, 1*2*4=8, 1*2*3=6）。

1. **初始化**：

   - n = 4
   - answer = [1, 1, 1, 1]

2. **第一遍遍历（前缀乘积，从左到右）**：

   - i=1: answer[1] = answer[0] _ nums[0] = 1 _ 1 = 1
   - i=2: answer[2] = answer[1] _ nums[1] = 1 _ 2 = 2
   - i=3: answer[3] = answer[2] _ nums[2] = 2 _ 3 = 6
   - 此时 answer = [1, 1, 2, 6]（分别代表左侧乘积：空=1, 1, 1*2=2, 1*2\*3=6）

3. **第二遍遍历（后缀乘积，从右到左）**：
   - right = 1（起始）
   - i=3: answer[3] _= right → 6 _ 1 = 6；然后 right _= nums[3] → 1 _ 4 = 4
   - i=2: answer[2] _= right → 2 _ 4 = 8；然后 right _= nums[2] → 4 _ 3 = 12
   - i=1: answer[1] _= right → 1 _ 12 = 12；然后 right _= nums[1] → 12 _ 2 = 24
   - i=0: answer[0] _= right → 1 _ 24 = 24；然后 right _= nums[0] → 24 _ 1 = 24（但循环结束）
   - 最终 answer = [24, 12, 8, 6]

通过这个案例，可以看到前缀提供左侧信息，后缀提供右侧信息，二者相乘即得结果。如果数组有零，例如 nums = [1, 0, 3, 4]，则输出 [0, 36, 0, 0]（因为包含零的位置乘积为零，其他位置需排除零）——代码会自动处理。

#### 另一个案例：包含零和负数 nums = [-1, 1, 0, -3, 3]

- 第一遍前缀：answer = [1, -1, -1, 0, 0]
- 第二遍后缀：
  - i=4: 0 \* 1 = 0, right=3
  - i=3: 0 _ 3 = 0, right=3_(-3)=-9
  - i=2: -1 * (-9) = 9, right=-9*0=0
  - i=1: -1 * 0 = 0, right=0*1=0
  - i=0: 1 _ 0 = 0, right=0_(-1)=0
- 输出：[0, 0, 9, 0, 0]（验证正确：排除自身后的乘积）。

这个算法高效且鲁棒，适合实际应用。如果需要测试更多案例，可以在 JS 环境中运行函数验证。

> 题目含义：
> 给你一个数组 `nums`，返回一个新数组 `answer`，其中
> `answer[i] = nums` 里**除 `nums[i]` 以外所有数的乘积**，
> 且**不能使用除法**，时间 `O(n)`，空间 `O(1)`（不算返回数组）。

## 一、先看一下代码结构

```js
var productExceptSelf = function (nums) {
  const n = nums.length;
  const answer = new Array(n).fill(1);

  // 第一遍：前缀乘积
  for (let i = 1; i < n; i++) {
    answer[i] = answer[i - 1] * nums[i - 1];
  }

  // 第二遍：后缀乘积（用一个变量 right）
  let right = 1;
  for (let i = n - 1; i >= 0; i--) {
    answer[i] *= right;
    right *= nums[i];
  }

  return answer;
};
```

核心思想就两句话：

1. 第一遍从左往右，先把“左边所有数的乘积”放进 `answer[i]`
2. 第二遍从右往左，用一个变量 `right` 负责“右边所有数的乘积”，再乘到 `answer[i]` 上

这样 `answer[i] = 左边所有数乘积 * 右边所有数乘积`，完美避开 `nums[i]` 自己。

---

## 二、案例一：`nums = [1, 2, 3, 4]`

我们一步一步算。

### 1️⃣ 初始化

```js
nums = [1, 2, 3, 4];
n = 4;
answer = [1, 1, 1, 1]; // 初始全是 1
```

---

## 三、第一遍：前缀乘积

这段代码：

```js
for (let i = 1; i < n; i++) {
  answer[i] = answer[i - 1] * nums[i - 1];
}
```

含义：

- `answer[i]` 里存的是：**`nums[0] * nums[1] * ... * nums[i-1]`**
- 也就是 `i` 左边所有元素的乘积（不包含 `i` 自己）

我们逐项算：

### 🔹 i = 1

```js
answer[1] = answer[0] * nums[0]
          = 1         * 1
          = 1
```

此时：

```js
answer = [1, 1, 1, 1];
```

（`answer[1]` 表示：索引 `1` 左边只有 `nums[0] = 1`，所以 `1`）

---

### 🔹 i = 2

```js
answer[2] = answer[1] * nums[1]
          = 1         * 2
          = 2
```

此时：

```js
answer = [1, 1, 2, 1];
```

解释：

- `answer[2] = nums[0] * nums[1] = 1 * 2 = 2`
- 也就是索引 `2` 左边的乘积：`1 * 2`

---

### 🔹 i = 3

```js
answer[3] = answer[2] * nums[2]
          = 2         * 3
          = 6
```

此时：

```js
answer = [1, 1, 2, 6];
```

解释：

- `answer[3] = nums[0] * nums[1] * nums[2] = 1 * 2 * 3 = 6`
- 即索引 `3` 左边的所有数的乘积：`1 * 2 * 3`

---

### ✅ 第一遍结束，`answer` 内容含义：

```js
answer[0] = 1; // 左边没数，记作 1（乘法单位元）
answer[1] = 1; // = nums[0]
answer[2] = 1 * 2; // = nums[0] * nums[1]
answer[3] = 1 * 2 * 3; // = nums[0] * nums[1] * nums[2]

answer = [1, 1, 2, 6];
```

---

## 四、第二遍：后缀乘积（从右往左）

这段代码：

```js
let right = 1;
for (let i = n - 1; i >= 0; i--) {
  answer[i] *= right;
  right *= nums[i];
}
```

含义：

- `right` 变量保存的是：**当前索引 i 右边所有数的乘积**
- `answer[i]` 原本有“左边的乘积”，再乘上 `right`（右边的乘积），就得到“除了自己以外所有数的乘积”。

我们一步步来。

### 初始

```js
right = 1;
answer = [1, 1, 2, 6];
```

---

### 🔹 i = 3（看 nums[3] = 4）

```js
answer[3] *= right; // answer[3] = 6 * 1 = 6
right *= nums[3]; // right = 1 * 4 = 4
```

此时：

```js
answer = [1, 1, 2, 6];
right = 4;
```

解释：

- 对于索引 `3`（元素 4），右边没有数 ⇒ 右侧乘积为 `1`（中性元素）
- 最终 `answer[3] = 左边乘积(1*2*3=6) * 右边乘积(1) = 6`

---

### 🔹 i = 2（看 nums[2] = 3）

```js
answer[2] *= right; // answer[2] = 2 * 4 = 8
right *= nums[2]; // right = 4 * 3 = 12
```

此时：

```js
answer = [1, 1, 8, 6];
right = 12;
```

解释：

- 此刻 `right = 4`，代表索引 `2` 右边所有数的乘积：`nums[3] = 4`
- `answer[2]` 原本是左边乘积：`1 * 2 = 2`
- 所以 `answer[2] = 左(1*2) * 右(4) = 8`，即：
  `nums[0] * nums[1] * nums[3] = 1 * 2 * 4 = 8`

---

### 🔹 i = 1（看 nums[1] = 2）

```js
answer[1] *= right; // answer[1] = 1 * 12 = 12
right *= nums[1]; // right = 12 * 2 = 24
```

此时：

```js
answer = [1, 12, 8, 6];
right = 24;
```

解释：

- 此刻 `right = 12`，代表索引 `1` 右边所有数的乘积：`nums[2] * nums[3] = 3 * 4 = 12`
- `answer[1]` 原本是左边乘积：`nums[0] = 1`
- 所以 `answer[1] = 左(1) * 右(3*4=12) = 12`
  即 `1 * 3 * 4 = 12`

---

### 🔹 i = 0（看 nums[0] = 1）

```js
answer[0] *= right; // answer[0] = 1 * 24 = 24
right *= nums[0]; // right = 24 * 1 = 24
```

此时：

```js
answer = [24, 12, 8, 6];
right = 24; // 已经没用了
```

解释：

- 此刻 `right = 24`，代表索引 `0` 右边所有数的乘积：`2 * 3 * 4 = 24`
- `answer[0]` 原本是左边乘积（左边没数，初始为 1）
- 所以 `answer[0] = 1 * (2*3*4) = 24`

---

## 五、最终结果

```js
nums = [1, 2, 3, 4];

answer = [24, 12, 8, 6];
// 对应：
// answer[0] = 2 * 3 * 4 = 24
// answer[1] = 1 * 3 * 4 = 12
// answer[2] = 1 * 2 * 4 = 8
// answer[3] = 1 * 2 * 3 = 6
```

✅ 完全符合题意。

## 六、再看一个带 0 的案例：`[1, 2, 0, 4]`

这个题最麻烦的是“有 0”时，用除法会很难处理，但前缀+后缀法自然搞定。

你可以自己推一遍，我帮你列好表格：

### 1️⃣ 第一遍后（前缀）

`nums = [1, 2, 0, 4]`

```js
answer 初始 = [1, 1, 1, 1]

// i = 1: answer[1] = answer[0] * nums[0] = 1 * 1 = 1
// i = 2: answer[2] = answer[1] * nums[1] = 1 * 2 = 2
// i = 3: answer[3] = answer[2] * nums[2] = 2 * 0 = 0

answer = [1, 1, 2, 0]
```

含义：

- `answer[0] = 1`
- `answer[1] = 1`
- `answer[2] = 1 * 2`
- `answer[3] = 1 * 2 * 0 = 0`

---

### 2️⃣ 第二遍（后缀）

```js
right = 1

i = 3:
  answer[3] *= right = 0 * 1 = 0
  right *= nums[3] = 1 * 4 = 4

i = 2:
  answer[2] *= right = 2 * 4 = 8
  right *= nums[2] = 4 * 0 = 0

i = 1:
  answer[1] *= right = 1 * 0 = 0
  right *= nums[1] = 0 * 2 = 0

i = 0:
  answer[0] *= right = 1 * 0 = 0
  right *= nums[0] = 0 * 1 = 0
```

最终：

```js
answer = [0, 0, 8, 0];
```

验证一下：

- 对于索引 0：`2 * 0 * 4 = 0`
- 对于索引 1：`1 * 0 * 4 = 0`
- 对于索引 2：`1 * 2 * 4 = 8`
- 对于索引 3：`1 * 2 * 0 = 0`

完美 ✅

---

## 七、整体思想总结

1. **不用除法**，靠“前缀乘积 + 后缀乘积”组合。
2. 用 `answer` 存前缀，用一个变量 `right` 存后缀，空间只用 `O(1)` 额外。
3. 两遍循环，每遍 `O(n)`，总时间 `O(n)`。
