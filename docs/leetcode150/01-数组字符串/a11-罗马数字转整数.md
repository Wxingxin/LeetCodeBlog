# 罗马数字转整数
## 题目

罗马数字包含以下七种字符: `I， V， X， L，C，D 和 M`。
```js
字符          数值
I             1
V             5
X             10
L             50
C             100
D             500
M             1000
```

例如， 罗马数字 2 写做 II ，即为两个并列的 1 。12 写做 XII ，即为 X + II 。 27 写做  XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。

但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。

同样地，数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：

- `I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。`

- `X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。 `

- `C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。`

给定一个罗马数字，将其转换成整数。

 

### 示例 1:

输入: `s = "III"`

输出: `3`
### 示例 2:

输入: `s = "IV"`

输出: `4`
### 示例 3:

输入: `s = "IX"`

输出: `9`
### 示例 4:

输入: `s = "LVIII"`

输出: `58`

解释: `L = 50, V= 5, III = 3.`
### 示例 5:

输入: `s = "MCMXCIV"`

输出: `1994`

解释: `M = 1000, CM = 900, XC = 90, IV = 4.`

## 

这题是 **LeetCode · 罗马数字转整数（Roman to Integer）**，典型的「**规则识别 + 遍历**」题，思路不复杂，关键是 **减法规则**。

---

## ✅ 核心思路（一遍遍历，最常用）

**正常情况**：
罗马数字是从左到右相加的

> 例：`LVIII = 50 + 5 + 1 + 1 + 1`

**特殊情况（减法规则）**：
如果 **当前字符的值 < 下一个字符的值**，说明这是减法：

> `IV = 5 - 1`
> `IX = 10 - 1`

所以我们可以这样做：

* 从左到右遍历字符串
* 如果 `当前值 < 后一个值` → **减去当前值**
* 否则 → **加上当前值**

---

## ✅ JS 标准解法（推荐，面试 / LeetCode 通吃）

```js
/**
 * @param {string} s
 * @return {number}
 */
var romanToInt = function(s) {
  // 1. 建立罗马数字到整数的映射表
  const map = {
    I: 1,
    V: 5,
    X: 10,
    L: 50,
    C: 100,
    D: 500,
    M: 1000
  };

  let res = 0;

  // 2. 从左到右遍历
  for (let i = 0; i < s.length; i++) {
    const cur = map[s[i]];
    const next = map[s[i + 1]]; // 最后一个 next 会是 undefined

    // 3. 判断是否触发减法规则
    if (next && cur < next) {
      res -= cur;
    } else {
      res += cur;
    }
  }

  return res;
};
```

## 🧠 例子拆解（非常重要）

### 示例：`s = "MCMXCIV"`

| 字符 | 当前值  | 下一个值 | 操作 | 累计   |
| -- | ---- | ---- | -- | ---- |
| M  | 1000 | 100  | +  | 1000 |
| C  | 100  | 1000 | −  | 900  |
| M  | 1000 | 10   | +  | 1900 |
| X  | 10   | 100  | −  | 1890 |
| C  | 100  | 1    | +  | 1990 |
| I  | 1    | 5    | −  | 1989 |
| V  | 5    | —    | +  | 1994 |

## ⏱️ 复杂度分析

* **时间复杂度**：`O(n)`（只遍历一遍字符串）
* **空间复杂度**：`O(1)`（映射表是常量）
